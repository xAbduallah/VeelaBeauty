# i18n Setup

## Quick Start

```tsx
"use client";
import { useTranslation } from "@/lib/i18n";

export default function MyComponent() {
  const { t } = useTranslation("common");
  return <h1>{t("header.login")}</h1>;
}
```

## Language Switcher

Already in Header. Users can switch EN/AR.

## Files

### Core Files

- **`lib/i18n/config.ts`** - Locales (en, ar), namespaces, default settings
- **`lib/i18n/client.ts`** - i18next initialization with browser detection
- **`lib/i18n/provider.tsx`** - React provider wrapping the app
- **`lib/i18n/hooks.ts`** - `useTranslation()`, `useLanguage()`, `translateNavigation()`
- **`lib/i18n/index.ts`** - Main exports

### Translation Files

```
lib/i18n/locales/
├── en/
│   ├── common.json      # Header, footer
│   └── nav.json         # Navigation items
└── ar/
    ├── common.json      # Header, footer
    └── nav.json         # Navigation items
```

### Components

- **`components/ui/LanguageSwitcher.tsx`** - Language dropdown
- **`app/layout.tsx`** - Wrapped with `<I18nProvider>`

## Usage Examples

### Basic

```tsx
const { t } = useTranslation("common");
<button>{t("header.login")}</button>
```

### Language Control

```tsx
const { currentLanguage, changeLanguage, isRTL } = useLanguage();
changeLanguage("ar");
```

### With Variables

```tsx
t("greeting", { name: "John" }) // "Hello John!"
```

## Adding Translations

1. Add key to both `lib/i18n/locales/en/[namespace].json` and `/ar/`
2. Use: `t("your.new.key")`

## Features

- ✅ EN/AR with RTL support
- ✅ Persistent language (localStorage)
- ✅ Browser detection
- ✅ TypeScript typed
